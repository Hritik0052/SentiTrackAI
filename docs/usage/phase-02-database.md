# Phase 2 — Database Layer

SQLAlchemy 2.0 engine/session + Alembic migrations. Works with SQLite (dev) or PostgreSQL (prod)
by only changing `DATABASE_URL`.

## What shipped
- Declarative `Base` + `TimestampMixin` — [`app/database/base.py`](../../app/database/base.py)
- Engine, `SessionLocal`, `get_db` dependency — [`app/database/session.py`](../../app/database/session.py)
- Alembic wired to app settings + models — [`alembic/env.py`](../../alembic/env.py)

## Switching databases

`.env`:
```env
# Dev (default, zero config)
DATABASE_URL=sqlite:///./sentitrack.db

# Production
DATABASE_URL=postgresql+psycopg://sentitrack:sentitrack@localhost:5432/sentitrack
```

The engine auto-adds `check_same_thread=False` for SQLite and uses batch-mode migrations so
Alembic `ALTER TABLE` works on SQLite too.

## Migration commands

```bash
# Create a new autogenerated migration (used from Phase 3 onward)
alembic revision --autogenerate -m "create users table"

# Apply migrations
alembic upgrade head

# Roll back one step / inspect
alembic downgrade -1
alembic current
alembic history
```

## `get_db` usage pattern

```python
from fastapi import Depends
from sqlalchemy.orm import Session
from app.database import get_db

@router.get("/example")
def handler(db: Session = Depends(get_db)):
    ...
```

## Verify

```bash
alembic upgrade head   # creates alembic_version table, succeeds

```
